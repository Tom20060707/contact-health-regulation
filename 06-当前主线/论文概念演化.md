# 论文概念演化：从 TC-PPO 到接触域一致性与稳定性

## 0. 当前结论

当前最推荐的论文定位是：

> 人形机器人接触域中的一致性与稳定性研究：用接触一致性作为崩溃前诊断信号，用 PPO 稳定机制改善接触域稳定性。

这不是简单把 `TC-PPO` 改名，而是论文主线经历了几次真实收敛。

## 1. 早期：TC-PPO / FTC

早期核心是：

```text
TC-PPO = 用接触时间一致性调节 PPO
FTC = 足部/肢体接触时间一致性指标
```

这一阶段解决的问题是：PPO 在高维人形控制中会因为接触模式不稳定而训练崩溃。

贡献雏形：

- 发现 reward 崩溃前，接触时序会先异常；
- 用 FTC/CTS 类指标做训练侧门控；
- 用 Mode A/B/C 描述门控行为。

局限：

- FTC 容易被理解成只看脚；
- 如果把 FTC 用来解释操作任务，会显得过度泛化；
- `1 - CV²` 公式过于像周期步态指标。

## 2. 中期：诊断框架

中期最重要的变化是：论文不再试图证明“所有任务都提升”，而是强调诊断价值。

Mode A/B/C 的意义变成：

- **Mode A**：接触退化可恢复，保护机制有效；
- **Mode B**：固定阈值或接触结构不匹配导致保护锁死/不可恢复；
- **Mode C**：接触稳定，保护不应触发，但任务可能仍然停滞。

这让负结果可以进入论文主线，而不是被藏起来。

## 3. 4 月 28 日：CTS-PPO

4 月 28 日的关键是命名收敛：

```text
CTS = Contact Timing Stability
CTS-PPO = Contact Timing Stability-Guided PPO
```

这个阶段的优点：

- 不再说 `Temporal Consistency` 这种太泛的概念；
- 不再只强调 foot；
- 把“高维控制灾难”和“接触/形态不稳定性”连起来。

但它仍有风险：

- 如果 CTS 仍然用 `1 - CV²` 表示，就无法解释非周期任务；
- kitchen、insert、push、truck 等任务不是简单周期接触；
- 因此 CTS 不能直接作为总理论。

## 4. 4 月 29 日：CSC 总框架

4 月 29 日的核心突破是承认：

> 周期性接触间隔一致性只是接触域一致性的一个特例。

因此引入：

```text
CSC = Contact-Surface Consistency
```

CSC 的思想是：

- 不强迫所有任务呈现周期步态；
- 比较当前接触事件分布与稳定参考分布；
- 根据任务接触面集合定义诊断对象；
- 允许准周期、阶段化、事件驱动任务采用不同接触描述。

这使得论文从单一 CTS 指标升级为接触域研究框架。

## 5. 一致性与稳定性的最终分工

最稳的写法是：

```text
一致性 = 诊断信号
稳定性 = 目标性质
```

具体含义：

- **接触一致性**：接触事件、接触面、接触阶段是否形成可解释、可重复、可比较的模式；
- **接触域稳定性**：策略训练和执行是否在接触密集交互中保持可靠，不发生崩溃或失控；
- **CSC/CTS**：用于发现退化；
- **CTS-PPO / calibrated PPO**：用于抑制退化并恢复稳定域。

## 6. 当前写作应避免的误区

不要写成：

- TC-PPO universally improves HumanoidBench；
- FTC/CTS 可以解释所有任务；
- `1 - CV²` 是所有接触任务的核心指标；
- kitchen 已被解决；
- hand/object/tactile CTS 已经实验验证。

应该写成：

- 当前实验验证了支撑/肢体接触稳定性诊断；
- CSC 是更一般的框架；
- 非周期任务暴露了旧指标边界；
- Mode B/C 是诊断发现，不是失败遮羞布；
- 操作任务需要任务相关接触面集合和未来实验扩展。
