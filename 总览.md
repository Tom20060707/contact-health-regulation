# 论文修改总览

## 一句话版本

这篇论文的主线已经从早期的 **TC-PPO 性能改进**，演化为：

> 人形机器人接触域中的一致性与稳定性研究：用接触一致性指标预警接触域退化，用 PPO 稳定机制抑制崩溃并支持后续任务推进。

## 为什么 README 不能再停在 4 月 16 日

旧 README 只覆盖 `2026-04-16` 到 `2026-04-27`，对应的是论文中期阶段：FTC、Mode A/B/C、soft gating、PBRS/Kitchen 边界。

但 `2026-04-28` 和 `2026-04-29` 发生了更关键的概念重构：

- `TC-PPO / FTC` 不再足以描述整篇论文；
- `CTS-PPO` 被提出，用于强调 Contact Timing Stability；
- 又进一步发现 `CTS = 1 - CV²` 不能覆盖非周期任务；
- 因此引入更上层的 CSC / Contact-Surface Consistency；
- 论文标题和主线转向“接触域一致性与稳定性”。

所以现在的仓库结构必须把“当前主线”放到前面，把旧 TC-PPO 材料放到历史演化中解释。

## 概念演化

### 阶段 1：TC-PPO / FTC 起点

最初论文关注：

- PPO 在 contact-rich humanoid tasks 中的训练不稳定；
- FTC 作为足部/肢体接触时间一致性指标；
- TC-PPO 用接触指标调整 PPO 更新；
- Mode A/B/C 用于解释门控何时有效、何时无效。

这一阶段的价值是提供了真实实验入口和最早的诊断框架。

### 阶段 2：诊断框架化

随后发现，论文不能写成“TC-PPO 在所有任务上提升性能”。更诚实的定位是：

- Mode A：接触退化可被门控保护并恢复；
- Mode B：固定阈值或接触结构不匹配导致保护锁死或不可恢复；
- Mode C：接触稳定但任务可能仍然没有进展。

这时负结果不再是失败，而是诊断框架的一部分。

### 阶段 3：CTS-PPO 与接触时序稳定性

4 月 28 日左右，命名和叙事进一步收束：

- `Temporal Consistency` 太泛；
- `Foot Contact-Time Consistency` 太窄；
- `GCTC / HTC / FTC` 缩写过多，容易显得随意；
- 更合适的表达是 `Contact Timing Stability`。

于是形成：

```text
CTS = Contact Timing Stability
CTS-PPO = Contact Timing Stability-Guided PPO
```

但这一版仍有一个风险：如果把 CTS 公式写成 `1 - CV²`，它仍然偏向周期步态，不适合所有非周期任务。

### 阶段 4：CSC 总框架

4 月 29 日的关键突破是承认：

> 接触间隔规律性不是所有接触任务的普适指标。

许多任务是非周期、阶段化或事件驱动的：

- kitchen；
- truck；
- push；
- insert；
- 短程加速；
- 楼梯平台切换。

因此当前更稳的总框架是：

```text
CSC = Contact-Surface Consistency
```

CSC 比较当前接触事件分布与稳定参考分布，不强迫所有任务都具有周期步态。旧 CTS/FTC 公式可以作为准周期支撑接触的轻量实例保留。

### 阶段 5：一致性与稳定性分工

当前最清楚的论文表达是：

```text
接触域研究
  |
接触域退化
  |----------------|
诊断信号           稳定机制
  |                 |
接触一致性          接触域稳定性
  |                 |
崩溃前预警          崩溃抑制 / 稳定恢复
  |                 |
CSC / CTS           CTS-PPO / calibrated PPO
```

简化说：

- 一致性是尺子；
- 稳定性是目标；
- CSC/CTS 是诊断；
- PPO 稳定机制是干预。

## 当前写作原则

- 不再把 TC-PPO 写成简单 performance booster。
- 不再把 FTC/CTS 写成所有任务通用指标。
- 不虚构 hand-object、object-environment、tactile CTS 实验。
- 允许把未验证内容写成 future work、extension、boundary analysis。
- 所有实验数字必须回到真实日志、图表脚本或 `main.tex`。
- 仓库中旧表述保留为演化过程，不代表最终论文口径。
