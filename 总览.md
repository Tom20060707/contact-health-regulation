# 项目总览

这个项目现在的主线是 **Contact-Health Regulation**：把人形机器人在复杂接触任务中的训练过程，看成一个可以诊断、预警、保护、增强和挽救的接触健康调节过程。

如果只用一句话说：

> 我想研究的是：PPO 在人形机器人接触任务里为什么会失稳，接触模式能不能比 reward 更早提醒我们训练正在变坏，以及能不能根据这个提醒让 PPO 更新得更稳。

## 1. 动机故事

人形机器人训练很难，不只是因为动作维度高。它一直在和外界接触：脚踩地、身体保持平衡、手推物体、末端执行器插入或碰撞。

这些接触一旦变乱，策略可能会慢慢滑向坏状态。更麻烦的是，reward 有时反应得比较慢；等 reward 明显掉下来，策略可能已经在错误的接触模式里待了一阵。

所以这个项目最开始想抓住一个提前量：

```text
接触模式先变坏
  ↓
reward 后面才崩
```

如果这个提前信号可靠，就可以在 PPO 还没被坏样本带偏太多时，让更新变得更保守。

## 2. 核心问题

这个项目不想简单回答“我的方法是不是所有任务都赢”。更重要的问题是：

1. 人形机器人训练中的接触退化能不能被量化？
2. 这种退化是否出现在 reward 崩溃之前？
3. 当接触状态变差时，PPO 应该如何调整更新？
4. 哪些任务适合这种接触诊断，哪些任务需要更丰富的任务阶段或物体状态建模？

这几个问题比一个单独算法名更重要。

## 3. 模型和方法

现在的方法可以分成三个层级。

### 接触诊断

先从机器人和环境的接触中构造状态信号。

早期主要看足部接触时间一致性，也就是 FTC / CTS。它适合上楼、行走、支撑和平衡恢复这类任务，因为这些任务的接触有一定节奏。

后来发现很多任务不是周期性的，比如 kitchen、insert、push、truck。于是主线变成更宽的接触面一致性：不只问接触间隔是不是均匀，而是问任务相关接触面是否还接近稳定参考模式。

### 接触健康状态

诊断信号不是最终目的，它用来判断训练状态。

现在更清楚的状态包括：

| 状态 | 含义 |
|---|---|
| normal | 接触和任务进展都没有明显异常 |
| warning | 接触状态开始变差，但还没到强干预 |
| protect | 接触退化明显，PPO 更新需要保守 |
| hold | 接触恢复稳定，先观察，不急着推动 |
| push | 接触稳定但任务进展停滞，可以小剂量增强探索 |
| rescue | 保护也救不回来时的部署安全扩展，不能写成已验证结果 |

### PPO 调节

当接触状态不健康时，PPO 不能继续像普通训练一样大步更新。

可用的机制包括：

- 软门控：接触越差，更新越保守；
- 自适应阈值：用 warm-up 分布确定不同任务自己的低阈值和高阈值；
- 连续恢复计数：不是一次恢复就退出保护，而是连续稳定后再退出；
- Push 剂量控制：稳定后如果进入平台期，可以小幅推动，但必须监控并及时退出。

## 4. 实验结果怎么看

目前实验更像机制验证，而不是简单追求所有任务平均分最高。

比较有价值的结果包括：

- 一些任务中，接触状态确实能解释 PPO 为什么开始不稳。
- soft gating 和自适应阈值让方法比早期固定硬阈值更合理。
- slide、powerlift 等任务中能看到稳定性、方差或回报上的改善。
- ablation 说明 hard gating、soft gating 和 vanilla PPO 的差别不是纯叙事。
- Mode A/B/C 帮助说明方法什么时候有效、什么时候只是看见问题但救不回来、什么时候接触指标本身不是任务瓶颈。

同时也有边界：

- stair、sit-hard 等任务不一定表现为均值显著提升，可能更多体现为稳定性或机制解释。
- basketball、insert-normal、truck 等任务提醒我们：接触稳定不等于任务成功。
- kitchen 一类非周期操作任务需要任务阶段、物体状态和更细接触建模，不能用足部接触时序硬套。

## 5. 名字为什么变了

早期名字叫 TC-PPO，意思是用接触时间一致性去调节 PPO。这个名字可以保留在历史记录里，但现在不适合当项目主标题。

原因很简单：现在研究问题已经比“一个 PPO 小改动”更宽。

更合适的关系是：

```text
TC-PPO：早期实现入口
FTC / CTS：足部或接触时序诊断指标
CSC：更宽的接触面一致性
CHR：Contact-Health Regulation，当前项目主线
CHS：Contact-Health State，训练状态变量
```

对外介绍时，建议直接讲 Contact-Health Regulation。旧名只用来解释研究演化，不放在第一屏。

## 6. 现在最该讲清楚的三部分

### 动机

人形机器人接触任务中，训练失败常常先表现为接触模式退化。reward 可能滞后，因此需要更早的诊断信号。

### 方法

用接触诊断信号判断训练处于 normal、warning、protect、hold、push 或 rescue 等状态，再根据状态调节 PPO 更新强度。

### 实验

实验说明这种接触健康视角在部分任务中能解释和改善训练稳定性，同时也暴露出非周期操作任务需要更一般的接触面和任务阶段建模。
