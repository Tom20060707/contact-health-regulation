## 当前论文主线

仓库早期以 **TC-PPO / FTC** 为中心，但后续主线已经发生明显演化。当前更准确的论文方向是：

> 研究人形机器人接触域中的一致性与稳定性：用接触一致性作为崩溃前诊断信号，用 PPO 稳定机制改善接触域稳定性。

概念演化大致是：

```text
TC-PPO / FTC
→ contact-rich PPO 失稳诊断
→ CTS-PPO / Contact Timing Stability
→ Contact-Domain Consistency and Stability
→ CSC 总框架 + CTS 支撑接触特例
```

因此，仓库中出现的 `TC-PPO`、`FTC`、`CTS-PPO`、`CSC` 不是互相矛盾，而是论文想法逐步收敛的历史痕迹。

## 最重要的结论

1. **TC-PPO 是历史起点，不是当前唯一表述。**  
   早期论文试图证明一个 PPO 变体在 HumanoidBench 上提升性能；后续逐步发现，更稳的定位是接触域退化诊断与稳定化。

2. **FTC/CTS 的价值不能写成“通用万能指标”。**  
   `1 - CV²` 这类接触间隔一致性公式更适合准周期支撑接触，不能直接解释 kitchen、insert、push、truck 等非周期或阶段化任务。

3. **当前更合理的总框架是 CSC + CTS。**  
   CSC 描述 contact-surface consistency 的总框架；CTS 是支撑接触或接触时序稳定性上的已验证实例。

4. **一致性与稳定性要分工。**  
   一致性是诊断信号，负责发现接触域退化；稳定性是目标性质，代表策略训练和执行是否不崩。

5. **Mode A/B/C 应解释为稳定域状态，而不是任务好坏分类。**  
   它们帮助区分可恢复退化、接触结构不匹配、稳定但停滞、门控漏检等情况。
