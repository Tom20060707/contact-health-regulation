# 实验结果和边界

这篇笔记只讲实验应该怎么对外解释。它不是最终数据表，具体数字写进论文前仍要回到 `main.tex`、图表脚本和原始训练日志核对。

## 先说实验想证明什么

这个项目的实验不应该被包装成“所有任务上都赢”。

更稳的目标是证明三件事：

1. 在一些 contact-rich humanoid tasks 中，接触状态确实能解释训练不稳定；
2. 当接触状态退化时，基于接触健康的 PPO 调节能减少继续恶化；
3. 方法有清楚边界：非周期操作任务、稀疏奖励任务和物体状态主导的任务，不能只靠足部接触时序解释。

这样讲更像科研，也更诚实。

## 已经比较有支撑的结论

### 1. 接触信号能解释部分 PPO 失稳

在上楼、支撑、平衡恢复等任务中，接触模式和训练稳定性之间有比较自然的关系。

当足部接触节奏突然变乱、支撑不稳定或接触窗口异常时，策略往往更容易进入坏状态。这说明 CTS / FTC 这类指标不是随便造出来的，它们确实抓住了一部分人形机器人训练中的关键现象。

### 2. 自适应阈值比固定阈值更合理

早期固定阈值容易出问题。不同任务的健康接触分数天然不一样，一个任务里的低分不一定代表另一个任务里的异常。

用 warm-up 分布定阈值更合理：

```text
tau_low  = P5(score_warmup)
tau_high = P75(score_warmup)
```

这让方法先学习“这个任务自己的正常接触水平”，再判断当前是否退化。

### 3. soft gating 比硬开关更稳

硬门控容易在阈值附近抖动。soft gating 把保护强度变成连续值，接触越差保护越强，恢复后再逐渐放开。

这更符合训练过程本身：接触健康不是二元开关，而是有程度的。

### 4. 部分任务上能看到稳定性或表现改善

当前记录里，slide、powerlift 等任务提供了比较有价值的证据：有的体现为平均回报提升，有的体现为方差下降或训练更稳。

这些结果适合支撑论文的主张：

> 接触健康调节可以在某些接触退化明显的任务中改善 PPO 训练过程。

但不要写成：

> 方法在所有任务上显著优于 PPO。

后者太满，也不符合现有实验。

## Mode A/B/C 应该怎么讲

Mode A/B/C 不是任务难度排名，也不是方法成败排行榜。

更合理的解释是训练状态和方法边界：

| Mode | 更稳的解释 |
|---|---|
| Mode A | 接触退化被诊断到，保护机制也确实帮助训练恢复 |
| Mode B | 接触退化被看见了，但现有调节没救回来，说明阈值、接触面或任务结构还不够 |
| Mode C | 接触指标没有明显报警，但任务也可能没完成，说明接触健康不是唯一瓶颈 |

这样写，Mode B/C 不是尴尬失败，而是帮助说明方法边界。

## 需要诚实承认的边界

### 1. 接触稳定不等于任务成功

Kitchen、insert、truck 这类任务提醒我们：接触看起来稳定，不代表任务真的完成。

比如机器人可能稳定地接触了物体，但没有把物体推到正确位置；或者接触没有崩，但任务阶段没有推进。

所以后续需要引入：

- 任务阶段；
- 物体状态；
- 手-物体接触；
- 物体-环境接触；
- 进度奖励或阶段成功信号。

### 2. 足部接触时序不是通用指标

CTS / FTC 对上楼、行走、支撑类任务有意义，但不能硬套到所有 manipulation 任务。

对外讲的时候要强调：

> CTS / FTC 是 Contact-Health Regulation 的一个支撑接触实例，不是全部理论。

### 3. Push 和 Rescue 不能写成已验证主结果

Push 是一个合理方向：当接触稳定但任务进展停滞时，可以小剂量推动探索。

Rescue 也是一个合理部署扩展：当 protect 救不回来时，可以考虑回滚 checkpoint 或更强干预。

但如果实验还没有完整验证，就只能写成扩展、讨论或下一步，不能把它们放得像核心已验证贡献。

## 给导师看的实验总结

可以这样讲：

> 实验目前不是想证明方法在所有 HumanoidBench 任务上都刷到最高分，而是验证“接触状态能否解释和调节 PPO 训练稳定性”。结果显示，在一些支撑接触或接触退化明显的任务中，接触健康信号和 soft gating 能带来稳定性或表现改善；同时，非周期操作任务暴露出仅靠足部接触时序不够，需要更一般的接触面、任务阶段和物体状态建模。这些边界反而帮助我把论文从一个单纯 PPO trick，收束成接触健康调节框架。

## 下一步实验最重要的事

1. 把所有关键数字回到真实训练日志和图表脚本核对。
2. 区分“已验证贡献”和“未来扩展”。
3. 对 Mode A/B/C 给出更清楚的任务例子。
4. 不再新增缩写，优先让动机、方法、实验闭环更清楚。
